Licensing

UMNi Systems™ is licensed as one connected product line delivered across three progressive tiers: UMNi-NTFY™, UMNi-ALRT™, and UMNi-CMND™. Licensing is structured per account, with each tier expanding the depth of notifications, awareness, and remote interaction available to the user.

One System. Three Tiers.

Rather than operating as separate unrelated tools, UMNi Systems™ is positioned as one connected software line with three access levels. Users can begin with lightweight notification delivery, move into deeper live awareness, and then progress into full remote command functionality as their requirements grow.

UMNi-NTFY™

Entry tier focused on core notification delivery and essential Telegram-based updates.

UMNi-ALRT™

Expanded awareness tier built for deeper monitoring, live event reporting, and broader trade environment visibility.

UMNi-CMND™

Full capability tier extending ALRT with remote command and interaction features.

License Basis

  • Licensing is structured on a per-account basis.
  • Each activated tier is intended for the account it is issued to.
  • Access level depends on the selected tier: NTFY, ALRT, or CMND.
  • Higher tiers build on the same core system rather than acting as separate products.

Activation & Access

  • Users receive access based on the selected tier and active licensing status.
  • UMNi-NTFY™ is available as the free entry level.
  • UMNi-ALRT™ and UMNi-CMND™ require an active paid subscription.
  • Feature access may expand or reduce in line with upgrades, renewals, or account status.

Upgrades

Users can move upward through the UMNi Systems™ tier structure as their workflow evolves. This means starting with UMNi-NTFY™, upgrading into UMNi-ALRT™, and later stepping into UMNi-CMND™ when remote command features are required.

  • NTFY → ALRT upgrade path supported
  • ALRT → CMND upgrade path supported
  • Tier progression is built into the product structure

Support & Setup

Support and setup are handled separately from the core license itself. This keeps the licensing structure clean while leaving room for optional onboarding, configuration help, and paid support where required.

  • Core license access is separate from setup assistance
  • Support can be offered independently from license tier
  • Setup requirements may vary depending on user environment and feature tier

Current Tier Structure

  • UMNi-NTFY™ — free entry level
  • UMNi-ALRT™ — paid awareness tier
  • UMNi-CMND™ — paid full access tier with remote features

Updates & Continuity

  • UMNi Systems™ may continue to evolve across all tiers over time.
  • Features, refinements, and structural changes may be introduced as the platform matures.
  • Current access remains tied to the user’s active tier and licensing status.
