What is UMNi-ALRT?

UMNi-ALRT is a lightweight monitoring layer for MT5 trading systems. It observes key trading events and pushes structured notifications directly to Telegram so traders can stay informed without needing to constantly watch charts.

Alerts include trade activity, system status notifications, heartbeat checks and end-of-session performance summaries.

Requirements

UMNi-ALRT is designed to run alongside MetaTrader and Telegram. For the current live range, MetaTrader 5 is the primary platform. If you are running a dedicated MetaTrader 4 build, use the matching MT4 version and keep the setup path aligned to that terminal.

  • MetaTrader 5 – required for the current live product release
  • MetaTrader 4 build – use only where a dedicated MT4 version has been supplied
  • Telegram – required for live message delivery
  • Stable internet connection – needed for platform and Telegram communication
  • Demo-first setup – recommended before any live deployment

Core capabilities

  • Real-time trade alerts – instant notifications when trades open or close
  • Session summaries – automated reporting of wins, losses and net results
  • Heartbeat monitoring – confirms the trading engine is running
  • Telegram notifications – alerts delivered instantly to mobile devices
  • Structured message format – alerts designed for clarity and quick analysis
  • Multi-account monitoring – track several trading systems from one channel

Alert types

UMNi-ALRT generates structured alerts across the full lifecycle of a trading session.

  • System online confirmation
  • Trade opened
  • Trade closed
  • Session summary
  • Ping / heartbeat checks

Lightweight by design

UMNi-ALRT does not execute trading logic itself. It focuses purely on monitoring and reporting, making it extremely stable and easy to integrate into existing trading setups.
