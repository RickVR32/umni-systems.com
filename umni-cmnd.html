What is UMNi-CMND?

UMNi-CMND is the command and control layer of the UMNi ecosystem. Rather than observing only, it is designed to let an operator send structured instructions into a supported trading environment.

It works as a more advanced companion to UMNi-ALRT, extending awareness into action. Where UMNi-ALRT reports what is happening, UMNi-CMND enables controlled interaction with the system when intervention is required.

Requirements

UMNi-CMND is designed to run alongside MetaTrader and Telegram. For the current live range, MetaTrader 5 is the primary platform. If you are running a dedicated MetaTrader 4 build, use the matching MT4 version and keep the setup path aligned to that terminal.

  • MetaTrader 5 – required for the current live product release
  • MetaTrader 4 build – use only where a dedicated MT4 version has been supplied
  • Telegram – required for live command delivery
  • Stable internet connection – needed for platform and Telegram communication
  • Demo-first setup – recommended before any live deployment

Core capabilities

  • Remote command delivery – send structured instructions into supported trading systems
  • Controlled trade actions – support open, close, pause, resume, and emergency-style actions
  • Live status visibility – combine command use with clear account and execution visibility
  • Telegram command flow – control access delivered through a familiar Telegram workflow
  • Structured response layer – governed interaction instead of loose or untracked commands
  • Advanced operator control – positioned above UMNi-ALRT as the higher-tier control product

Command types

UMNi-CMND supports structured control across the key points of a live trading workflow.

  • System online confirmation
  • Live status requests
  • Trade open actions
  • Trade close actions
  • Command list and control menu access

Built for controlled intervention

UMNi-CMND does not replace the execution environment. It adds a structured remote control layer so you can respond, intervene, and manage supported systems without needing to stay at the terminal full time.
